# Project Rules

- Do not use Expo or Capacitor.
- Do not use Firebase, Supabase, Auth0, Vercel, or Netlify.
- Do not deploy or connect to AWS unless explicitly approved in a future task.
- Keep business logic independent of React Native and AWS.
- Do not use floating-point values for financial currency calculations.
- Do not add incomplete money or invoice-calculation logic without a dedicated design task.
- Android is the only native verification target for this milestone; web is first-class.
- Services and infrastructure are documentation-only placeholders until implementation tasks begin.
