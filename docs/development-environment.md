## Metro and pnpm workspace notes

The original generated Metro configuration used the React Native default only:

```js
module.exports = mergeConfig(getDefaultConfig(__dirname), {});
```

That configuration did not include the repository root in `watchFolders` or explicitly search both app-local and repository-root `node_modules`. With pnpm, `apps/mobile/node_modules/@babel/runtime` is a junction into the repository root pnpm store, so Metro could fail to follow the target and returned HTTP 500 while resolving `@babel/runtime/helpers/interopRequireDefault`.

Node resolution confirmed the dependency was present and resolvable at:

```text
node_modules/.pnpm/@babel+runtime@7.29.7/node_modules/@babel/runtime/helpers/interopRequireDefault.js
```

Metro now keeps `getDefaultConfig(projectRoot)` and adds only:

- `watchFolders: [workspaceRoot]`
- `resolver.nodeModulesPaths` with `apps/mobile/node_modules` first and repository-root `node_modules` second

No `disableHierarchicalLookup`, broad React/React Native aliases, shameful hoisting, or third-party Metro resolver was added.
